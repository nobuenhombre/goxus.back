# Goxus

[![Go Version](https://img.shields.io/badge/Go-1.26.1-00ADD8?logo=go)](https://go.dev)
[![License](https://img.shields.io/badge/License-Apache%202.0-blue)](LICENSE)

**Goxus** is a Go backend application with dependency injection (Google Wire), a Gin HTTP API server, cron job scheduling, and PostgreSQL code generation via xo. Designed for production deployment as a systemd service on Linux.

---

## Quick Start

```bash
# Build the binary
cd service/deployments/goxus/linux
make build-app

# Run in default mode
./bin/goxus/linux/goxus

# Run as HTTP service
./bin/goxus/linux/goxus -runtype=service -config=configs/local/config.yaml

# Check version
./bin/goxus/linux/goxus --version

# Regenerate Wire DI (after provider changes)
make wire
```

---

## Features

- **3 run modes** — `init`, `service` (HTTP + cron), and default (business logic)
- **Dependency injection** via Google Wire — clean provider/cleanup pattern
- **Gin HTTP server** with graceful shutdown (SIGINT/SIGTERM)
- **Cron job scheduler** — YAML-configurable schedule per job
- **YAML configuration** — multi-environment presets (local, develop, production)
- **PostgreSQL code generation** — xo with custom Go templates
- **systemd integration** — install, start, stop, restart, upgrade as a service
- **Static binary** — CGO_ENABLED=0, fully static linux/amd64 build
- **Log file support** — optional log file with standard logger redirect

## Architecture

```
src/
  cmd/goxus/                  # Entrypoint
    main.go                   #   Panic recovery, --version fast path, Wire init
    app.go                    #   IApp orchestrator, run-mode switch
    wire.go                   #   Wire injector (build tag: wireinject)
    wire_gen.go               #   Auto-generated by `make wire`
  internal/
    app/goxus/
      cli/                    # CLI flag parsing
      config/                 # YAML app configuration (load/save)
      domain/                 # Business logic service
      log/                    # Log file management
      api/server/             # Gin HTTP server
        router/               #   Routes, handlers, middlewares
      cron-job/               # Cron scheduler
        jobs/example/         #   Example cron job
    pkg/
      db/goxus/               # xo-generated PostgreSQL types
  scripts/xo/                 # DB tooling (codegen, migrations, backups)
configs/                      # Multi-environment configs + systemd units
  local/                      #   Local development
  develop/                    #   Development server
  production/                 #   Production
service/deployments/          # Build & deployment Makefiles
bin/                          # Compiled binaries
```

### DI dependency chain

```
main()
  └─ initializeApp() [Wire]
       ├─ ProvideCLI()        → cli.Service
       ├─ ProvideConfigApp()  → configapp.Service      [depends on CLI]
       ├─ ProvideDomain()     → DomainService           [depends on CLI + Config]
       ├─ ProvideLogFile()    → ILogFile                [depends on CLI]
       ├─ ProvideAPI()        → IHTTPServer             [depends on Config + Log + Domain]
       ├─ ProvideExampleJobs()→ *cron.Cron              [depends on Domain]
       └─ newApp()            → IApp                    [top-level orchestrator]
```

Every provider returns `(Service, func(), error)` — Wire tracks cleanup in reverse construction order.

## Run Modes

| Mode      | Flag                     | Behaviour                                                    |
|-----------|--------------------------|--------------------------------------------------------------|
| `init`    | `-runtype=init`          | Placeholder init (default)                                   |
| `service` | `-runtype=service`       | Start cron scheduler + HTTP server, graceful shutdown        |
| default   | (no flag or unrecognized)| Run `domain.Run()` (business logic placeholder)              |

### CLI flags

```
-runtype=init|service    Run mode (default: init)
-config=<path>           Path to YAML config file (default: config.yaml)
-log=<path>              Path to log file (optional, empty = stdout)
--version                Print version and exit
```

## Configuration

### Config file structure

```yaml
hosts:
  api:
    host: 127.0.0.1      # Bind address
    post: "8080"         # Bind port (note: "post" is intentional — see Gotchas)
cron:
  example_job:
    enabled: true
    schedule: '@every 10m'
```

| Environment | Config file | systemd unit |

| Develop | `configs/develop/config.yaml` | `api_goxus_dev.service` |
| Production | `configs/production/config.yaml` | `api_goxus.service` |

## Build & Deploy

### Development (root Makefile)

```bash
make wire          # Regenerate wire_gen.go
make deps          # Re-init go.mod and fetch dependencies
make help          # List available commands
```

### Production build (service/deployments/goxus/linux/)

```bash
make build-app           # Static linux/amd64 binary
make build-app-progress  # Build with progress bar (requires gawk)
make install-app         # Symlink to /usr/local/bin/goxus
make uninstall-app       # Remove symlink
make install-service     # Enable systemd unit (set SERVER_ROLE)
make uninstall-service   # Disable systemd unit
make service-start       # systemctl start api_goxus
make service-stop        # systemctl stop api_goxus
make service-restart     # systemctl restart api_goxus
make service-status      # systemctl status api_goxus
make upgrade             # stop → build → start → status
```

## Database

PostgreSQL code generation uses [xo](https://github.com/xo/xo) with custom Go templates. Database migrations use [golang-migrate/migrate](https://github.com/golang-migrate/migrate) CLI (`migrate`).

```bash
# Regenerate DB types from the database schema
cd src/scripts/xo/
./xo.sh goxus/xo.yaml

# Migrations (golang-migrate)
cd src/scripts/xo/goxus/
./migrate-new.sh           # Create a new migration
./migrate-up.sh            # Apply pending migrations
./migrate-down.sh          # Rollback last migration
```

## Conventions

- **Error handling:** Wrap errors with `ge.Pin(err)` from `suikat/pkg/ge`
- **DI providers:** Every `ProvideXxx` returns `(Service, func(), error)` — cleanup is never nil
- **Version:** Single source of truth in `version/version.go` (SemVer)
- **Testing:** Table-driven tests with YAML fixtures
- **Git keepfiles:** `.gitkeep` files in every directory to preserve structure in version control

---

## Gotchas

- **Port YAML key:** The config uses `post:` (typo) not `port:`. The struct tag in `config-server.go` is `yaml:"post,omitempty"`. Changing it would break all environment config files.
- **Wire cleanup:** All providers must return a non-nil cleanup. Cleanup runs in reverse provider order.
- **CGO_ENABLED=0:** Build forces CGO off for static linking. If you need C libraries, update `go-build.mk`.
- **GOSUMDB=off:** Sum database disabled in Makefile. Do not re-enable without understanding the tradeoff.
- **Root vs deploy Makefile:** The root `Makefile` is minimal (wire + deps). Full build/deploy lives in `service/deployments/goxus/linux/Makefile`.
- **xo-generated code:** `src/internal/pkg/db/goxus/xo_db.xo.go` is auto-generated — edit templates, not the output.
- **No tests yet** for domain, CLI, and server packages — only `config-app_test.go` exists.
- **Go proxy:** Uses `proxy.golang.org,direct` — set `GOPROXY=off` for air-gapped builds.
