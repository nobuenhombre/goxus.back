# Goxus

## Overview

- **Purpose:** Backend HTTP API server with cron job scheduler, CLI-configurable multi-mode application (init/service/default).
- **Domain:** Generic backend service — PostgreSQL via xo codegen, Gin HTTP, cron tasks.
- **Module:** `goxus`
- **Version:** v0.1.0 (SemVer, defined in `src/internal/app/goxus/version/version.go`)
- **License:** Apache 2.0

## DI & Wire

- Every package under `internal/app/goxus/` has a `provider.go` with `ProviderSet` and 1-2 `ProvideXxx` functions.
- Every provider returns `(Service, func(), error)` — the cleanup function tears down in **reverse construction order** (Wire tracks this automatically).
- The injector lives in `src/cmd/goxus/wire.go` (build tag `wireinject`), generates `wire_gen.go`.
- Run `make wire` to regenerate after changes to providers or dependencies.

### Dependency chain

```
main()
  └─ initializeApp() [wire]
       ├─ ProvideCLI()           -> cli.Service          [cli/provider.go]
       ├─ ProvideConfigApp()     -> configapp.Service    [depends on cli.Service]
       ├─ ProvideDomain()        -> DomainService        [depends on cli + config]
       ├─ ProvideLogFile()       -> ILogFile             [depends on cli]
       ├─ ProvideAPI()           -> IHTTPServer          [depends on config + log + domain]
       ├─ ProvideExampleJobs()   -> *cron.Cron           [depends on domain]
       └─ newApp()               -> IApp                 [top-level orchestrator]
```

## Run Modes (CLI)

The app has 3 run modes, controlled by `-runtype` flag:

| Mode      | Flag                | Behaviour                                               |
|-----------|---------------------|---------------------------------------------------------|
| `init`    | `-runtype=init`     | Placeholder init logic (logs + returns)                 |
| `service` | `-runtype=service`  | Starts cron scheduler + Gin HTTP server, graceful shutdown on SIGINT/SIGTERM |
| default   | (no flag or others) | Runs `domain.Run()` (business logic)                    |

### CLI flags

```
-runtype=init|service    (default: init)
-config=config.yaml      (path to YAML config, default: config.yaml)
-log=<path>              (path to log file, optional)
--version                print version and exit (fast path before Wire init)
```

## Conventions

- **Error wrapping:** `ge.Pin(err)` from suikat/pkg/ge consistently.
- **DI pattern:** Each provider returns `(Service, func() error, error)`. The cleanup func is always populated (even if a no-op) to satisfy Wire's `Cleanup` signature.
- **Type assertions:** Providers assert `cli.Service.(*cli.Config)` to extract concrete type — this is the established pattern, not ideal but consistent.
- **Version:** Single source of truth in `version/version.go`.
- **Testing:** Table-driven tests in `config-app_test.go`. YAML test fixtures (`_load.yaml`, `_save.yaml`) live alongside the source.
- **Timestamps:** Все временные метки — только UTC. `time.Now().UTC()` при записи в БД, в SQL — `(NOW() AT TIME ZONE 'UTC')` вместо `NOW()`, `make_interval(days => $1)` вместо `$1 || ' days'`. Подробнее: скилл `go-timestamp-utc`.

## Gotchas

- **Wire cleanup chaining:** Cleanup functions are called in reverse provider order. If a mid-chain provider fails, all earlier cleanups are called correctly by generated code. Never pass a nil cleanup to Wire.
- **Port YAML key:** The config YAML uses `post:` (typo) for the port field. This is intentional/legacy — changing it would break all config files (`config/config-server.go` has `yaml:"post,omitempty"`).
- **CGO_ENABLED=0:** Build forces CGO off for static linking (see `go-build.mk` and deployment Makefile).
- **NO tests for domain, CLI, server yet** — only `config-app_test.go` exists.
- **GOSUMDB=off:** Sum database is disabled in Makefile; verification relies on go.sum content. Don't remove this without understanding the tradeoff.
- **Go proxy:** Uses `proxy.golang.org` with `direct` fallback (see Makefile).
- **Secondary Makefile:** The root `Makefile` is minimal (wire + deps). Full build/deploy is in `service/deployments/goxus/linux/Makefile`.
- **xo generated code:** `src/internal/pkg/db/goxus/xo_db.xo.go` is auto-generated — do not edit by hand. Regenerate via `src/scripts/xo/xo.sh <yaml-file>`.

## Deployment
- Binary: statically linked linux/amd64, installed to `/usr/local/bin/goxus`
- systemd service: `configs/{local,develop,production}/api_goxus*.service`
- Logs: `/var/log/goxus/api.log` (api_goxus.service configures `-log=/var/log/goxus/api.log`)
- Working directory: `/opt/goxus/configs/` (for config file resolution)
- Cron jobs are enabled/disabled via YAML config (`cron.example_job.enabled`)
